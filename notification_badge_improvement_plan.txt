NOTIFICATION BADGE PLACEMENT IMPROVEMENT PLAN
===============================================
Date: 2026-04-15

PROBLEM:
  When a coach comments on a video, the notification badge appears on the
  "More" tab in the bottom toolbar. Tapping "More" clears the visual cue,
  leaving the athlete stranded with no trail to the actual feedback. The
  badge should guide athletes directly to the coach's folder and the
  specific video with new feedback.

CURRENT STATE (what already works):
  - ActivityNotificationService already tracks:
    * unreadCountByFolder: [String: Int] — per-folder unread counts
    * unreadVideoIDs: Set<String> — specific videos with unread feedback
    * unreadFolderVideoCount: Int — total unread across all folders
  - AthleteFoldersListView FolderRow already shows per-folder unread badges
  - CoachVideoCard and AthleteVideoRow already show "New" badges on videos
  - CoachVideoPlayerView marks video as read when opened (markVideoRead)
  - AthleteFolderDetailView marks folder as read when opened (markFolderRead)

  So the drill-down badges exist — they're just hidden behind the More tab.

WHAT NEEDS TO CHANGE:

========================================
CHANGE 1: Add badge to Dashboard coach card (HIGH IMPACT)
========================================
File: Views/Dashboard/DashboardView.swift

The athlete dashboard already has a coach videos card. It reads
`activityNotifService.unreadFolderVideoCount` but may not prominently
display it as a badge.

Action:
  - Ensure the coach/shared folders card on the Dashboard shows a red
    badge with the total unread count (e.g., "3 new" or just the count)
  - This is the athlete's home screen — most visible spot
  - Tapping this card should navigate to AthleteFoldersListView (or
    directly to the folder if there's only one)

========================================
CHANGE 2: Move tab badge from More to Home/Dashboard tab
========================================
File: Views/Navigation/MainTabView.swift

Current: MoreTabBadgeModifier (lines 684-696) puts the badge on More tab
Action:
  - Remove or reduce the More tab badge for coach feedback
  - Add the unread coach feedback count as a badge on the Home/Dashboard
    tab instead, since that's where the coach card lives
  - The Home tab badge tells athletes "open your dashboard, there's
    something new" — then the dashboard card guides them deeper

Implementation:
  - Add a new badge modifier to the Home tab:
      .badge(activityNotifService.unreadFolderVideoCount)
  - Keep the Shared Folders row badge in More as a secondary path
  - Consider: if unreadFolderVideoCount > 0, the More tab badge
    could still show it as a secondary indicator, but the primary
    badge should be on Home

========================================
CHANGE 3: Per-folder badges on the Dashboard coach card
========================================
File: Views/Dashboard/DashboardView.swift (or the coach card component)

If the dashboard shows individual folder rows or coach names:
  - Show per-folder unread count next to each folder/coach name
  - Use activityNotifService.unreadCountByFolder[folderID] for counts
  - This lets athletes see "Coach Smith: 2 new" at a glance

If the dashboard only shows an aggregate card:
  - Show the total count on the card
  - When tapped, navigate to AthleteFoldersListView where per-folder
    badges already exist

========================================
CHANGE 4: Ensure AthleteFolderDetailView does NOT auto-clear badges
========================================
File: AthleteFoldersListView.swift (lines 399-402)

Current: markFolderRead() is called in .task when the folder detail opens,
which clears ALL unread badges for the entire folder at once — even if the
athlete hasn't looked at every video.

Action:
  - REMOVE the markFolderRead() call from AthleteFolderDetailView's .task
  - Instead, rely on per-video markVideoRead() in CoachVideoPlayerView
  - This way, the folder badge decrements as the athlete watches each
    video, giving proper "breadcrumb" tracking
  - The folder row badge in AthleteFoldersListView will show the
    remaining unread count accurately

This is the most important behavioral fix — without it, opening a folder
once clears all badges even if you didn't watch any videos.

========================================
CHANGE 5: Badge on Games/Lessons sub-tabs within folder detail
========================================
File: AthleteFoldersListView.swift (AthleteFolderDetailContent) or
      CoachFolderDetailView.swift

If the folder detail view has sub-sections (Games vs Lessons):
  - Show unread count on each section header or tab
  - Filter unreadVideoIDs by the videos in each section
  - e.g., "Games (2 new)" and "Lessons (1 new)"

This requires knowing which videos are games vs lessons. If the video
metadata includes a type/category field, filter unreadVideoIDs against
the videos in each section.

========================================
CHANGE 6: (Optional) Notification deep link to folder
========================================
File: MainTabView.swift (notification routing, lines 348-354)

Current: Push notifications may already route to the shared folder via
  Notification.post(navigateToSharedFolder, folderID)

Action:
  - Verify that tapping a push notification for a coach comment navigates
    to the specific folder (not just the More tab)
  - If not, update PushNotificationService action handling to post the
    navigateToSharedFolder notification with the correct folderID
  - Ideal: deep link all the way to the video
    (navigateToSharedFolder → folder → auto-scroll to video)
